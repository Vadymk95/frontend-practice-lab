# Story 6.4: PWA Update Notification

Status: done

## Story

As a **user**,
I want to be notified when a new version of the app is available,
so that I can apply the update immediately without stale content persisting.

## Acceptance Criteria

1. **Given** a new version has been deployed (service worker detects changed precache manifest)
   **When** the user opens the app
   **Then** a non-blocking toast appears at the bottom: "New version available. Refresh?"
   **And** the toast does not interrupt an active session

2. **Given** the user taps "Refresh" in the update toast
   **When** `skipWaiting()` is called
   **Then** the new service worker activates and the page reloads with the latest version

3. **Given** the user dismisses the toast
   **When** dismissal occurs
   **Then** the current session continues uninterrupted
   **And** the toast does not reappear until the next app open

## Dev Notes

### BLOCKER: Story 6.3 Must Complete First

This story depends on `vite-plugin-pwa` being installed and configured (Story 6.3). `useRegisterSW` is exported from `vite-plugin-pwa/react` — that package must exist.

### useRegisterSW API

```typescript
// From vite-plugin-pwa/react
import { useRegisterSW } from 'virtual:pwa-register/react';

const usePwaUpdateToast = () => {
    const {
        needRefresh: [needRefresh],
        updateServiceWorker
    } = useRegisterSW({
        onRegistered(r) {
            console.log('[SW] registered', r);
        }
    });

    const [dismissed, setDismissed] = useState(
        () => sessionStorage.getItem('pwa_update_dismissed') === '1'
    );

    const isVisible = needRefresh && !dismissed;

    const handleUpdate = () => {
        void updateServiceWorker(true); // true = reload page after SW activation
    };

    const handleDismiss = () => {
        setDismissed(true);
        sessionStorage.setItem('pwa_update_dismissed', '1');
    };

    return { isVisible, handleUpdate, handleDismiss };
};
```

**`needRefresh`** is `[Ref<boolean>, Setter]` — destructure as `[needRefresh]` only (you don't need the setter).

### TypeScript: virtual:pwa-register Module

`virtual:pwa-register/react` is a virtual module generated by `vite-plugin-pwa`. TypeScript needs to know about it:

```typescript
// vite-env.d.ts or src/vite-env.d.ts — add:
/// <reference types="vite-plugin-pwa/client" />
```

If `vite-env.d.ts` doesn't exist, create it in `src/` or add the reference to `src/vite-env.d.ts`.

### registerType: 'prompt' is Required

`vite.config.ts` must have `registerType: 'prompt'` (not `'autoUpdate'`). `'autoUpdate'` would bypass our toast entirely. Verify this is set in Story 6.3's implementation before running this story.

### Toast UI Pattern

```tsx
// PwaUpdateToast.tsx
export function PwaUpdateToast() {
    const { isVisible, handleUpdate, handleDismiss } = usePwaUpdateToast();

    if (!isVisible) return null;

    return (
        <div
            role="status"
            aria-live="polite"
            className="fixed bottom-4 left-4 right-4 z-50 flex items-center justify-between rounded-lg border border-border bg-background p-4 shadow-lg md:left-auto md:right-4 md:w-80"
        >
            <span className="text-sm text-text-primary">{t('pwa.updateAvailable')}</span>
            <div className="flex gap-2">
                <button onClick={handleUpdate} className="...">
                    {t('pwa.refresh')}
                </button>
                <button onClick={handleDismiss} aria-label={t('pwa.dismiss')} className="...">
                    ×
                </button>
            </div>
        </div>
    );
}
```

### No Toast Library

Project has no toast/sonner library (confirmed in Story 5.4 notes). Use the same fixed-position `div` pattern. Do NOT install a toast library.

### Mounting Point

Render `<PwaUpdateToast />` in the root layout component (e.g., `src/components/layout/AppShell.tsx` or directly in `App.tsx`) so it's always mounted regardless of current route. This ensures it shows during an active session without being tied to a specific page.
