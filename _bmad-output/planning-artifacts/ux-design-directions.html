What is the difference between useMemo and useCallback?
Why this matters useMemo(() => compute(), [deps]) returns the computed value and skips recomputation when deps unchanged. useCallback(() => fn, [deps]) returns the same function reference — critical when passing callbacks to memoized children to avoid breaking React.memo.

What does React.memo do, and when should you NOT use it?
React.memo does a shallow comparison. Avoid it when props contain new object/array references each render (it will always re-render anyway) or when the component is cheap to render.

When does useEffect run, and how does the dependency array change its behaviour? List all three modes.
Explanation
No array = runs after every render (expensive, often wrong). Empty array [] = runs once after mount. Array with deps = runs after mount and whenever any dep changes. The cleanup function in the return value runs before the next effect or on unmount.
